## Notes

¹ **EC → Covariance (test 14):** Python uses `scipy.optimize.brentq` (xtol=10⁻¹⁵), R uses `uniroot` (tol ≈ 1.22 × 10⁻⁴). Python finds a *more precise* root:  
- Python max |cov_to_ec(df, root) − target_ec| = **1.11 × 10⁻¹⁵**  
- R max |cov_to_ec(df, root) − target_ec| = **1.12 × 10⁻⁴**  
- Max |py_root − r_root| = 2.59 × 10⁻⁵  

Both roots are valid inverse solutions; the test verifies functional equivalence rather than bitwise root equality.

### Architecture Decisions

- **R subprocess enforcement:** Clustering (hclust) and the L-BFGS-B optimizer are executed via R subprocess calls to guarantee identical numerical behaviour. No scipy/Python fallback exists — a missing R installation raises `RuntimeError`.
- **Binary data transfer:** Simulation data and optimizer inputs use raw binary files (`writeBin`/`readBin` in R, `numpy.tofile`/`fromfile` in Python) to eliminate CSV parser rounding differences (~26% of doubles differ by 1 ULP between R's `read.csv` and Python's `pandas.read_csv`).
- **Reference data:** Stored in `tests/reference_data/` — CSV files with `sprintf("%.17e")` precision for floating-point values, plus `.bin` files for binary transfer.
