# Reference Data for CI Validation

This directory contains reference outputs from the R pipeline, run on small grids,
used to validate that the Python port produces numerically equivalent results.

## How to regenerate

Run the R pipeline with `resolution=10, n_sim=10, seed=42` (parameters.R test config):

```r
source("parameters.R")  # uses resolution=10
source("functions.R")
set.seed(42)
a_sim <- sim_expt_2d_nonstat(X, Y, df_true, alpha_true, a_matrix, b_matrix, g_matrix, n_sim=n_sim)
write.csv(as.data.frame(as.table(a_sim)), "simulation_10x10_seed42.csv", row.names=FALSE)
# ... then run c2–c5 and export results similarly
```

| File | Description | Generated by |

| `local_optimizer_selected_inputs.csv` | Lossless `(zi, zj, xl, yl)` arrays for the 5 selected local-MLE cells | `tests/generate_r_reference.R` |
| `local_optimizer_selected_outputs.csv` | Exact R-selected optimizer outputs, bounds, and metadata for those 5 cells | `tests/generate_r_reference.R` |
| `local_estimates_10x10.csv` | Local parameter estimates (a, b, γ) per grid cell | `c2_locest_calc.R` + `c3_locest_merge.R` |
| `clusters_locest_10x10.csv` | Cluster labels from ellipse-based method | `c4_locest_clust.R` |
| `clusters_saunders_10x10.csv` | Cluster labels from Saunders madogram method | `c5_saunders_clust.R` |
| `dist_helper_test_cases.csv` | Frozen `dist_x` / `dist_y` helper outputs from R | `tests/generate_r_reference.R` |

## Tolerance

Python tests assert `np.allclose(python_result, r_reference, atol=1e-6)`.
Small differences are expected due to:
- Different PRNG implementations (R vs NumPy)
- Floating-point ordering in optimisation
- Latin Hypercube starting point differences

For simulation, we compare **distributional properties** (marginal quantiles, extremal coefficients)
rather than exact values, unless both use the same seed and PRNG stream.
